Transparency and Accountability

Your algorithm should satisfy due process and accountability best practices for human-centered prediction.

Data Give users control over their data

What data has been collected about an individual?

Users should be able to examine in total what has been collected about them.

Error Correction

Users should be able to easily request that erroneous data be corrected.

Opt Out

Consider allowing users to turn off some data collection, and specify which data will still be collected after opt out.

Inference Give users control over the inferences made about them

When is prediction/personalization used?

Users should understand where and when prediction is taking place.

What inferences have been made about an individual?

Users should be able to see all inferences that have been made about them - including inferences that may be seen as "creepy" or undesirable.

What led to each decision?

Give a ranking of human understandable attributes that led to this decision. This may not be possible for several reasons:

  • Concerns over gaming of the system: Consider giving coarser information regarding the features that were important for this inference
  • Intellectual property concerns: Consider giving coarser information regarding the features that were important for this inference
  • Lack of interpretability of the model in use: Consider using a more interpretable model for human-centered prediction

Error Correction

Users should be able to mark inferences as incorrect. These can then be reviewed by humans and the corrected data can be incorporated as training data.

Opt Out

Consider allowing users to turn off prediction/personalization. Be clear about how this will impact users, for example in an inspection, will they be allocated to the highest risk?

Data Empowerment

Even if the algorithm is not in error, provide a set of suggestions to help this person if an undesirable inference has been made about them.
